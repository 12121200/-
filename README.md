**1.** Операционная система (ОС) — это набор программ, управляющий аппаратными ресурсами компьютера и обеспечивающий работу приложений и пользователя. Её назначение — эффективное распределение ресурсов, обеспечение многозадачности, безопасности и удобного интерфейса.

**2.** Основные функции: управление процессами, памятью, файловой системой, устройствами ввода-вывода, безопасность, организация интерфейса пользователя, сетевые функции.

**3.** Виды ОС: десктопные (Windows, Linux Mint), серверные (Linux, Windows Server), мобильные (Android, iOS), реального времени. По пользователям — однопользовательские и многопользовательские. По задачам — однозадачные и многозадачные.

**4.** Разрядность — количество бит, обрабатываемых одновременно (32 или 64 бит). 64-битная ОС поддерживает больший объём памяти и даёт выше производительность в современных задачах.

**5.** Ядро — центральная часть ОС, работающая в привилегированном режиме. Оно напрямую управляет оборудованием, процессами, памятью и устройствами.

**6.** Дистрибутив Linux — это полноценная ОС на базе ядра Linux плюс пользовательские программы, оболочки, пакетный менеджер и графическая среда (Ubuntu, Fedora, Debian). Ядро — только базовая часть.

**7.** Виды лицензий: проприетарные (закрытые — Windows, macOS) и свободные (open source — GPL, MIT). Свободное ПО позволяет изучать, изменять и распространять код.

**8.** Графический интерфейс (GUI) упрощает работу обычным пользователям. ОС может работать без него в режиме командной строки (особенно серверы).

**9.** Порядок загрузки: включение → BIOS/UEFI → POST → загрузчик (GRUB) → ядро → init/systemd → запуск служб и интерфейса.

**10.** BIOS — базовая система ввода-вывода, прошивка на материнской плате. Выполняет самотестирование (POST) и инициализацию оборудования перед загрузкой ОС.

**11.** UEFI — современная замена BIOS. Поддерживает GPT, Secure Boot, большие диски, быстрее работает и имеет графический режим.

**12.** CMOS — энергонезависимая память для хранения настроек BIOS/UEFI (время, порядок загрузки). Питается от батарейки.

**13.** POST — самотестирование оборудования при включении. Неисправности: звуковые сигналы, отсутствие изображения, зависание.

**14.** BOOT-меню вызывается клавишами F12, F10, Esc или F9. Используется для выбора устройства загрузки (флешка, диск).

**15.** Загрузочный сектор — первый сектор диска. Загрузчик (bootloader) — программа (GRUB, Windows Boot Manager), которая загружает ядро ОС.

**16.** Контроллер — микросхема, управляющая устройством. Plug-and-Play — автоматическое обнаружение и настройка устройств.

**17.** Прерывания — сигналы от устройств процессору. ОС обрабатывает их через специальные обработчики.

**18.** Виды памяти: регистры, кэш (L1-L3), оперативная (RAM), виртуальная, постоянная (SSD/HDD).

**19.** Оперативная память (RAM) — место для хранения кода и данных выполняющихся программ.

**20.** Виртуальная память позволяет использовать диск как расширение RAM. На Linux — swap-раздел/файл, на Windows — pagefile.sys.

**21.** Процесс — выполняющаяся программа с выделенным адресным пространством, ресурсами и состоянием. Программа — статический файл на диске.

**22.** Состояния процесса: новый, готов, выполнение, ожидание, завершён. ОС управляет ими через планировщик.

**23.** Физический диск — реальное устройство. Раздел — логическая часть диска. Логический диск — том с точкой монтирования или буквой.

**24.** Файловая система организует хранение, поиск, доступ к файлам, управляет правами и метаданными.

**25.** Windows: FAT32 (простая, совместимая, лимит 4 ГБ), exFAT (большие файлы, флешки), NTFS (журналирование, права, шифрование).

**26.** Linux: ext4 (основная, журналируемая, быстрые проверки, большие тома), XFS, Btrfs.

**27.** Файл — именованная область данных. Типы: обычные, каталоги, устройства, сокеты. Операции: создание, чтение, запись, удаление, копирование, изменение прав.

**28.** Монтирование — подключение файловой системы к дереву каталогов. Точка монтирования — каталог, куда подключается раздел.

**29.** Корневая директория (`/`) — начало всего дерева каталогов Linux.

**30.** Дерево каталогов Linux: `/bin`, `/sbin` — программы; `/etc` — конфиги; `/home` — данные пользователей; `/var` — логи; `/dev` — устройства; `/proc`, `/sys` — информация ядра.

**31.** `/bin` — основные утилиты; `/sbin` — системные; `/lib` — библиотеки; `/etc` — конфигурационные файлы.

**32.** `/home` — пользовательские данные; `/tmp` — временные файлы; `/var` — переменные данные; `/dev` — устройства; `/proc` — процессы; `/sys` — системная информация.

**33.** Командная оболочка (shell) — программа, интерпретирующая команды пользователя и передающая их ядру (bash, zsh).

**34.** Команда — название действия; ключ/опция — модификатор (`-l`); аргумент — объект (`/home`).

**35.** `man` — просмотр руководства по командам (`man ls`).

**36.** Работа с директориями: `pwd`, `cd`, `ls`, `mkdir`, `rmdir`.

**37.** Создание, копирование, перемещение, удаление: `touch`, `cp`, `mv`, `rm`, `mkdir`. Тип файла — `file`.

**38.** `cat` — вывод содержимого; `tac` — в обратном порядке; `head/tail` — начало/конец; `more/less` — просмотр; `strings` — строки из бинарных файлов.

**39.** Учётная запись — данные пользователя. UID — ID пользователя, GID — ID группы.

**40.** `/etc/passwd` — пользователи; `/etc/shadow` — пароли; `/etc/group` — группы.

**41.** `whoami` — текущий пользователь; `id` — UID/GID; `groups` — группы; `su` — смена пользователя; `sudo` — права root.

**42.** `useradd/usermod/userdel` — управление пользователями; `groupadd/groupmod/groupdel` — группами.

**43.** `r` — чтение, `w` — запись, `x` — выполнение. `chmod` — права, `chown` — владелец, `chgrp` — группа.

**44.** Режимы vi/vim: Normal (командный), Insert (вставка), Visual (выделение), Command-line (`:`).
